## 设计理念

Veloform v4.2.0 的交互设计遵循 **自然、即时、克制** 的原则，让用户在无感知的情况下完成任务。所有交互模式均基于 shadcn/ui + @base-ui/react 组件库实现，确保一致性和可访问性。

### 核心原则

1. **即时反馈** - 用户操作后 ≤100ms 内给出视觉反馈
2. **自然运动** - 动画模拟物理规律，使用缓动曲线
3. **克制使用** - 避免过度动画和装饰性效果
4. **清晰状态** - 每个交互状态都有明确的视觉表达
5. **可访问性** - 支持键盘导航、屏幕阅读器、Reduced Motion

## 1. 交互模式库

### 1.1 导航模式

#### 顶部导航栏 (Navbar)

**设计要点**：

- 固定顶部，背景半透明 + backdrop-blur
- Logo 左侧，导航链接居中/右侧
- 移动端：汉堡菜单 + Sheet 侧边抽屉
- Hover：微妙的颜色变化

**交互细节**：

- 导航链接 Hover：颜色变化（150ms）
- 当前页面：primary 色
- 移动端菜单：从右侧滑入（Sheet 组件）

**代码位置**：`src/components/layout/Navbar.tsx`

---

#### 面包屑 (Breadcrumb)

**设计要点**：

- 显示当前位置层级
- 分隔符：`/ ` 或 `>`
- 当前页面：加粗，无链接
- 父级页面：链接，hover 下划线

#### 分页

---

### 1.4 搜索模式

#### 实时搜索

**设计要点**：

- 输入时即时搜索（debounce 300ms）
- 清空按钮：输入框右侧 X 图标
- 搜索结果：下拉列表或页面跳转

**交互细节**：

- Focus：输入框边框变 primary 色
- 输入：实时显示结果（300ms debounce）
- 清空：点击 X 图标，清空并聚焦输入框

## 2. 反馈规范

### 2.1 悬停反馈 (Hover)

**设计原则**：所有可交互元素必须有悬停反馈

| 元素类型     | Hover 效果           | 时长  |
| ------------ | -------------------- | ----- |
| 按钮（填充） | 背景色加深，阴影增强 | 150ms |
| 按钮（描边） | 背景色变浅           | 150ms |
| 按钮（幽灵） | 背景色出现           | 150ms |
| 卡片         | 阴影增强 + 轻微上移  | 200ms |
| 链接         | 颜色变化或下划线     | 150ms |
| 列表项       | 背景色变化           | 100ms |

---

### 2.2 按下反馈 (Active/Press)

**设计原则**：提供按下的物理感

| 元素类型 | Active 效果                     | 时长  |
| -------- | ------------------------------- | ----- |
| 按钮     | `scale(0.98)` + translateY(1px) | 100ms |
| 卡片     | `scale(0.99)`                   | 100ms |
| 列表项   | 背景色加深                      | 100ms |

---

### 2.3 焦点反馈 (Focus)

**设计原则**：清晰的键盘导航焦点指示器

| 元素类型 | Focus 效果                            | 时长  |
| -------- | ------------------------------------- | ----- |
| 按钮     | `ring-3 ring-ring/50`                 | 150ms |
| 输入框   | `border-ring` + `ring-3 ring-ring/50` | 150ms |
| 链接     | 下划线 + 颜色变化                     | 150ms |
| 卡片     | `ring-2 ring-primary/20`              | 150ms |

**全局 Focus 样式**：

```css
:focus-visible {
  outline: none;
}
```

所有 shadcn/ui 组件均使用 `focus-visible:ring-3 focus-visible:ring-ring/50` 统一焦点样式。

---

### 2.4 加载反馈 (Loading)

**设计原则**：让用户知道系统在运行

#### 按钮加载

- 显示 Spinner，隐藏按钮文字
- 按钮禁用，cursor: wait
- Spinner 尺寸：16px（sm）、20px（default）、24px（lg）

**组件**：`lucide-react/Loader2` + `animate-spin`

---

#### 页面加载

- 全屏 Loading Screen
- 品牌 Logo + Spinner
- 可选：进度条（不确定进度）

**组件**：`src/components/ui/LoadingScreen.tsx`

---

#### 局部加载

- 卡片/区域加载：Skeleton 骨架屏
- Spinner 居中显示
- 背景半透明遮罩

**组件**：`src/components/ui/skeleton.tsx`

---

### 2.5 Toast 通知

**设计原则**：非侵入式的临时反馈

#### 类型

| 类型    | 图标          | 颜色 | 使用场景                       |
| ------- | ------------- | ---- | ------------------------------ |
| Success | CheckCircle   | 绿色 | 操作成功（保存、删除）         |
| Error   | XCircle       | 红色 | 操作失败（网络错误、验证失败） |
| Warning | AlertTriangle | 橙色 | 需要注意                       |
| Info    | Info          | 蓝色 | 信息提示（新功能、更新）       |
| Loading | Loader2       | 灰色 | 进行中操作                     |

#### 交互

- 自动消失：3-5 秒后自动关闭
- 手动关闭：右侧 X 按钮
- 悬停暂停：鼠标悬停时暂停自动关闭

**位置**：

- 桌面端：右上角
- 移动端：顶部居中

**组件**：`src/components/ui/sonner.tsx`（基于 Sonner）

## 3. 错误处理

### 3.1 输入错误

**设计原则**：即时、清晰、友好

#### 验证时机

- 即时验证：输入时实时验证（debounce 300ms）
- 提交验证：提交时验证所有字段

#### 错误提示

- 位置：输入框正下方
- 颜色：红色（`--error`）
- 字号：12px（sm）
- 图标：红色感叹号（可选）

**组件样式**：所有 shadcn/ui 表单组件支持 `aria-invalid` 状态，自动显示错误样式。

---

### 3.2 网络错误

**设计原则**：清晰的错误信息 + 重试选项

#### 错误处理组件

- 图标：WiFi 离线图标
- 标题："网络连接失败"
- 描述："请检查你的网络连接，然后重试"
- 操作：重试按钮

---

### 3.3 错误边界 (Error Boundary)

**设计原则**：优雅地捕获和处理运行时错误

#### 错误边界组件

- 标题："出现了一些问题"
- 描述："我们已记录此错误，将尽快修复"
- 操作：重新加载页面、返回首页
- 开发模式：显示错误堆栈

**组件**：

- `src/components/ui/error-boundary.tsx`
- `src/components/ui/ErrorBoundary.tsx`

## 5. 对话框与抽屉

### 5.1 对话框 (Dialog)

**设计要点**：

- 居中显示，背景遮罩
- 标题 + 描述 + 内容 + 操作按钮
- 点击遮罩或按 Esc 关闭
- 焦点管理：打开时聚焦第一个可交互元素

**动画**：

- 进入：fade-in + zoom-in-95（100ms）
- 退出：fade-out + zoom-out-95（100ms）

**组件**：`src/components/ui/dialog.tsx`

---

### 5.2 确认对话框 (Alert Dialog)

**设计要点**：

- 用于破坏性操作确认
- 不可通过点击遮罩关闭
- 明确的取消/确认按钮

**组件**：`src/components/ui/alert-dialog.tsx`

---

### 5.3 侧边抽屉 (Sheet)

**设计要点**：

- 从屏幕边缘滑入
- 常用于移动端导航、筛选面板
- 支持上下左右四个方向

**动画**：slide-in 动画

**组件**：`src/components/ui/sheet.tsx`

---

### 5.4 弹出层 (Popover)

**设计要点**：

- 相对于触发元素定位
- 轻量的内容展示
- 点击外部关闭

**组件**：`src/components/ui/popover.tsx`

---

### 5.5 下拉菜单 (Dropdown Menu)

**设计要点**：

- 垂直菜单列表
- 键盘导航支持
- 分组和分隔线

**组件**：`src/components/ui/dropdown-menu.tsx`

## 7. 无障碍规范 (WCAG 2.1 AA)

### 7.1 键盘导航

- ✅ 所有交互元素可通过 Tab 键访问
- ✅ 焦点顺序符合逻辑
- ✅ 焦点指示器清晰可见
- ✅ 支持方向键导航（菜单、列表）
- ✅ 所有 shadcn/ui 组件内置键盘支持

---

### 7.2 ARIA 属性

| 元素     | ARIA 属性                          | 说明           |
| -------- | ---------------------------------- | -------------- |
| 按钮     | `aria-label`                       | 图标按钮的描述 |
| 对话框   | `role="dialog"`, `aria-modal`      | 对话框角色     |
| 下拉菜单 | `aria-expanded`, `aria-haspopup`   | 展开状态       |
| 输入框   | `aria-invalid`, `aria-describedby` | 验证状态       |
| 加载     | `aria-busy`, `aria-live`           | 加载状态       |

---

### 7.3 色彩对比度

- ✅ 正文文本：≥ 4.5:1（WCAG AA）
- ✅ 大文本（18px+）：≥ 3:1
- ✅ 图标：≥ 3:1

---

### 7.4 动画偏好

**设计原则**：尊重用户的动画偏好设置

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

**代码位置**：`src/app/globals.css`（Reduced Motion 支持）
