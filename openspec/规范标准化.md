# 文档标准化规范

> **路径**: `/openspec/规范标准化.md`  
> **版本**: v3.4.1  
> **更新日期**: 2026-05-05

## 概述

本文档定义了 Veloform 项目规范文档体系的编写标准，涵盖文档格式、版本管理、命名规范和维护流程。所有 openspec 目录下的文档必须遵循本规范。

---

## 文档文件头规范

### 必需元素

每个规范文档**必须**在文件最顶部包含以下标准头部：

```markdown
# <文档标题>

> **路径**: `<文件相对路径>`  
> **版本**: v<MAJOR.MINOR.PATCH>  
> **更新日期**: YYYY-MM-DD

## 概述

<1-2 句简短说明本文档的目的和范围>
```

### 版本号规范

遵循 **SemVer 2.0.0** 标准：

| 组成部分 | 规则 | 示例 |
|---------|------|------|
| **MAJOR** | 主版本 - 不兼容的结构性变更 | v3.0.0 |
| **MINOR** | 次版本 - 向后兼容的功能新增 | v3.1.0 |
| **PATCH** | 修订版本 - 向后兼容的错误修正 | v3.1.1 |

**版本递增规则**：
- 文档结构重新组织 → 递增 MINOR 或 MAJOR
- 新增章节但不影响现有内容 → 递增 MINOR
- 错别字修正、格式调整 → 递增 PATCH
- 内容删除或结构变更 → 递增 MINOR 或 MAJOR

### 文档末尾规范

每个规范文档**必须**在文件末尾包含：

```markdown
---

**文档路径**: `<文件相对路径>`  
**最后更新**: YYYY-MM-DD  
**版本**: v<MAJOR.MINOR.PATCH>
```

## 标题层级规范

使用标准 Markdown 标题层级，**最多使用到 H3**：

```markdown
# H1 - 文档主标题（每个文件只有一个）

## H2 - 一级章节

### H3 - 二级子章节

#### H4 - 仅在复杂文档的详细列表中使用
```

**规则**：
- `#` 与标题文字之间有空格
- 避免跳级（如 H1 后直接 H3）
- 章节标题简洁明了，不超过 50 字符

## 代码块规范

### 语言标识

所有代码块**必须**标注语言：

```markdown
```typescript
// 正确示例
const example = 'code';
```

// 错误示例 - 无语言标识
const example = 'code';
```
```

### 常用语言标识

| 语言 | 标识 |
|------|------|
| TypeScript/JavaScript | `typescript` / `javascript` |
| JSON | `json` |
| CSS | `css` |
| HTML | `html` |
| Bash/Shell | `bash` / `shell` |
| Markdown | `markdown` |
| YAML | `yaml` |

### 代码注释规范

- 代码注释使用单行注释 `//`
- 不使用多行注释 `/* */` 作为文档
- 示例代码应可运行

## 表格规范

### 基本格式

```markdown
| 列1 | 列2 | 列3 |
|------|------|------|
| 内容 | 内容 | 内容 |
```

### 对齐规则

- 表头行使用 `|------|------|------|` 表示居左
- 内容行文字居左
- 数字列可使用 `| ---- |` 右对齐
- 重要列可使用 `|:----|` 左对齐强调

### 表格标题

每个表格前应有简短描述或标题：

```markdown
### 用户认证配置

| 参数 | 类型 | 说明 |
|------|------|------|
| ... | ... | ... |
```

---

## 列表规范

### 无序列表

```markdown
- 第一项
- 第二项
  - 子项（缩进 2 个空格）
  - 子项
- 第三项
```

### 有序列表

```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 任务列表

```markdown
- [ ] 未完成的任务
- [x] 已完成的任务
```

```markdown

## 版本历史规范

### 文档内版本历史

每个文档的版本历史**仅记录结构性变更**，日常内容更新不记录：

```markdown
### 版本历史

| 版本 | 日期 | 变更 |
|------|------|------|
| v1.1.0 | 2026-05-05 | 新增第 3 章性能监控内容 |
| v1.0.0 | 2026-05-01 | 初始版本 |
```

### 版本历史表位置

- 文档内版本历史 → 文档末尾
- 项目级版本历史 → `SPEC.md` 和 `openspec/README.md`

## 文档维护规范

### 审查频率

| 文档类型 | 审查频率 |
|----------|----------|
| 架构规范 | 重大版本发布时 |
| API 规范 | API 变更时 |
| 开发规范 | 每季度 |
| 安全规范 | 每半年 |
| 性能规范 | 每季度 |

### 贡献流程

1. 发现文档问题 → 创建 Issue 或直接提交 PR
2. 遵循本规范格式
3. 确保示例代码可运行
4. 更新相关文档的交叉引用
